## Goal
Update the `llm` package to support the advanced reasoning and thinking capabilities of next-generation LLMs (DeepSeek-V4-Pro and GPT-5-Mini) via OpenRouter, and ensure the test suite accurately reflects these changes.

## What happened
1. **Added Reasoning Parameters:** Updated `types.GenerateRequest` and the internal `openAIRequest` struct to include `ReasoningEffort` (for GPT-5 models) and `Thinking` (for DeepSeek models).
2. **OpenAI Provider Update:** Modified `provider_openai.go`'s `BuildRequest` method to map the new reasoning parameters into the request body. If a `Thinking` configuration is provided but `ReasoningEffort` is missing, it intelligently maps `Thinking.Type == "enabled"` to `ReasoningEffort: "high"` for cross-compatibility.
3. **OpenRouter Provider Update:** Updated `provider_openrouter.go`'s `BuildRequest` method to pass through both `ReasoningEffort` and `Thinking` configurations to the underlying models.
4. **Environment Configuration:** Extended `config.go` to support `LLM_REASONING_EFFORT`, `LLM_THINKING_TYPE`, and `LLM_THINKING_BUDGET` environment variables, automatically applying them to requests in `HTTPClient.Generate`.
5. **Test Suite Fixes:** Refactored `Provider.BuildRequest` to use `context.Context` and `*GenerateRequest` across all implementations to support cancellation and cleaner parameter passing. Fixed 5 broken test calls in `provider_test.go` and updated `mockProvider` in `client_test.go` to match the new signature.
6. **Max Tokens Regression Fix:** Addressed a silent regression in `provider_openai.go` where `MaxCompletionTokens` was being sent as 0 for non-reasoning models.
7. **OpenRouter Max Tokens Fix:** Removed `max_completion_tokens` from OpenRouter requests to prevent routing errors with incompatible backends, restoring the original test assertion.

## What went wrong
- The initial update to `Provider.BuildRequest` broke the test suite because `provider_test.go` and `client_test.go` were still using the old 5-parameter signature.
- Code review identified a silent regression where `max_completion_tokens` was mishandled for OpenAI and erroneously sent to OpenRouter. Both issues were quickly resolved and verified.

## What to watch for
- To enable DeepSeek's thinking mode, set `LLM_THINKING_TYPE="enabled"` and (optionally) `LLM_THINKING_BUDGET`.
- To enable GPT-5-Mini's reasoning mode, set `LLM_REASONING_EFFORT="high"`.
- These features are highly recommended for complex autonomous flows to reduce hallucination and improve DOM interpretation.

## Key decisions
- Opted to use `*GenerateRequest` in the `Provider` interface instead of adding more arguments to an already long function signature, making future parameter additions much simpler.
- Handled the OpenRouter `max_completion_tokens` quirk explicitly to ensure broad compatibility across backend providers.
