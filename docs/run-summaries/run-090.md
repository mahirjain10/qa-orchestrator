## Goal
Fix 4 logic errors identified in the codebase: (1) Context & goroutine leak from `defer llmCancel()` inside an iteration loop. (2) Complete bypass of retry limits — `RecoveryActionReplan` never counted toward `MaxRetries`. (3) Context cancellation ignored in browser runtime — Playwright calls hang for full 30s timeout even when cancelled. (4) Prompt injection via unsanitized DOM text in planner observation formatting.

## What happened

### Bug 1: Context & Goroutine Leak in Engine Loop
- `packages/agents/engine/engine.go` — `runAutonomousFlow()` line 402-403.
- `defer llmCancel()` inside the `for stepCount < maxSteps` loop means the cancel function never executes until the entire function returns. Each iteration created a new `context.WithCancel` and a new goroutine (from `autonomousLLMContext`'s lifecycle watcher) that stays alive for the entire flow duration. A 20-step flow leaked 20 goroutines + 20 contexts.
- **Fix:** Replaced `defer llmCancel()` with immediate `llmCancel()` call right after `GenerateNextStep` returns. The LLM context is only needed during step generation — once the step is generated, the context and its goroutine can be cleaned up immediately.

### Bug 2: Complete Bypass of Retry Limits (Infinite Loop)
- `packages/agents/recovery/recovery.go` — `ShouldRetry()` and `ShouldEscalate()` (lines 169-181), and `packages/agents/engine/engine.go` — `RecoveryActionReplan` case (line 660).
- `ShouldRetry` and `ShouldEscalate` only checked for `RecoveryActionRetry`. But `isLocatorError`/`isTimeout` errors trigger `RecoveryActionReplan`, not `Retry`. Since Replan never counted, the agent looped infinitely: failure → Replan → backoff → try again → failure → Replan → ...
- Additionally, the `RecoveryActionReplan` case in `runAutonomousFlow` did not increment `result.Retries`, so even if the recovery functions were fixed, the retry counter would stay at 0.
- **Fix (recovery.go):** Both `ShouldRetry` and `ShouldEscalate` now accept `RecoveryActionReplan` as a retryable action.
- **Fix (engine.go):** Added `result.Retries++` to the `RecoveryActionReplan` case in the autonomous flow switch.

### Bug 3: Context Cancellation Ignored in Browser
- `packages/browser-runtime/runtime.go` — all methods in both `BrowserRuntime` and `FlowBrowserRuntime`.
- Every method checked `ctx.Err()` *before* calling Playwright, but the Go context was never passed to the Playwright API. If the user pressed 'x' in the TUI while Playwright was blocking (e.g., `page.Goto` with 30s timeout), the call would hang until the Playwright driver timeout elapsed because `ctx.Err()` had already passed.
- **Fix:** Added a `runWithContext` helper that executes the Playwright call in a goroutine and selects on both the result channel and `ctx.Done()`. If the context is cancelled mid-call, the function returns `ctx.Err()` immediately.
- Applied to all 8 methods in both `BrowserRuntime` and `FlowBrowserRuntime`: `Navigate`, `Click`, `Fill`, `WaitForSelector`, `TextContent`, `InnerHTML`, `Evaluate`, `Screenshot`.

### Bug 4: Prompt Injection via Unsanitized DOM Text
- `packages/agents/planner/planner.go` — `formatObserveUIObservation()` line 145.
- Raw DOM text (`tag`, `selector`, `text`, `class`, `warning`, `pageState`) was directly interpolated into the LLM prompt using `%v` format strings. A malicious website could include elements with text/classes like `<div class="Ignore all previous instructions...">` to inject commands into the system prompt context.
- **Fix:** Added a `sanitizeDOM()` helper that strips null bytes (`\x00`), markdown code fences (`` ``` ``), and angle brackets (`<`, `>` → `&lt;`, `&gt;`). Applied to all DOM-sourced values in the observation formatting function.

## What to watch for
- `llmCancel()` is now called immediately after `GenerateNextStep` returns, not deferred. If any code path between the cancel and the end of the iteration body references `llmCtx`, it will get a cancelled context. This is safe because `GenerateNextStep` is the only consumer of `llmCtx`.
- `ShouldRetry` and `ShouldEscalate` now treat `RecoveryActionReplan` identically to `RecoveryActionRetry`. Any test that expects Replan to bypass retry limits will need updating.
- The `runWithContext` goroutine pattern creates a short-lived goroutine for each Playwright call. On context cancellation, the goroutine still completes in the background (Playwright finishes, writes to buffered channel, exits) — no leak.
- `sanitizeDOM` converts `<` and `>` to HTML entities. The LLM prompt will show `&lt;div&gt;` instead of `<div>` for elements that contain angle brackets in their text/class/tag values. Normal values (e.g. `<div>` shown as tag) are unaffected since the tag value `div` doesn't contain angle brackets.

## Key decisions
- **Bug 1:** Immediate cancel after `GenerateNextStep` is cleaner than tracking exit paths. The LLM context is ephemeral — no reason to keep it alive after the step is generated.
- **Bug 3:** Used a goroutine+channel pattern instead of setting shorter Playwright timeouts based on context deadlines. This works universally (contexts without deadlines too) and is consistent with how Go handles cancellation.
- **Bug 4:** Sanitization (escape) was chosen over rejection (skip element). If we skip elements with dangerous text, the LLM loses visibility into the page. Escaping preserves information while preventing injection.
