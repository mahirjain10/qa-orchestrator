# Run 051 — V4 Phase 10: Status Bar + Contextual Help

**Date:** 2026-05-20
**Agent:** qwen3.6-plus-free
**Phase:** V4 TUI Revamp — Phase 10

## Goal
Replace the static footer with a contextual status bar that shows relevant keys based on active view and state.

## Changes

### `apps/tui/internal/screens/main.go`

**Enhanced `renderStatusBar()`:**
- Hides on terminals shorter than 20 rows.
- Left side: run status badge (e.g., `RUNNING`) + truncated run ID (12 chars max), or "IDLE" if no run.
- Right side: contextual key bindings via new `contextualKeys()` method.
- Message line: shows recent message (within 5 seconds) above the bar using `style.Msg`.
- Uses `style.BgDark` background, `style.Dim` for keys.
- Calculates dynamic spacer between left and right sections based on terminal width.

**Added `contextualKeys()`:**
- Dashboard + run: `space:pause  x:cancel  s:steer  ?:help`
- Dashboard + idle: `enter:select  r:refresh  ?:help`
- Traces: `/:filter  S:failures  F:follow  ?:help`
- Flows: `enter:detail  r:retry  k:skip  ?:help`
- Default (Report): `?:help`

**Deleted:** Old static footer/message rendering — the previous `renderStatusBar()` only showed the message line. Now fully replaced with the spec-compliant implementation.
