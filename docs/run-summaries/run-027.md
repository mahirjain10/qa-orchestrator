## What went wrong
- Previous implementation bypassed the TUI's clean exit by trapping `SIGINT` manually, leaving the user's terminal in a broken state.
- Stores were instantiated but dropped before reaching the `MainScreen`.

## What to watch for
- Avoid using `os.Exit()` inside background goroutines when running a TUI application, as it prevents terminal restore sequences from firing.
- Ensure all required dependencies (stores) are actually injected into the constructors that need them.

## Key decisions
- Let bubbletea naturally handle interrupt signals (`Ctrl+C`) to guarantee terminal restoration.
- Added a `SetMessage` helper in `MainScreen` to display the startup message natively within the TUI UI loop.
