4. **T9 — cmd/main.go:446-505 cancelled race:** After the second `cancelled` check (line 488-490) released `pauseMu`, there was a TOCTOU window before semaphore acquire (line 497). Fixed by re-checking `cancelled` under `pauseMu` immediately after semaphore acquire, releasing the semaphore slot if cancelled.

## What to watch for
- `CalculateDelay` now uses a value receiver; any caller that relies on it being a pointer receiver will break.
- Wait/semaphore patterns must always re-check cancellation after acquire — the TOCTOU window between unlock and acquire is real.

## Key decisions
- **Return by value** for `DefaultRetryConfig` rather than a pointer getter. Prevents mutation without allocating on every call (small struct).
- **Filter fallbacks** instead of using a set data structure — simple, efficient for small lists.
- **Post-semaphore check** for cancellation rather than holding `pauseMu` across the semaphore acquire (which could block indefinitely holding the lock).
