## What to watch for
- `Close()` should be called when the engine is done to clean up the `lifecycleCtx` goroutine. Currently no existing code calls it — but since the goroutine's purpose is to bridge lifecycle cancellation to a Go context, and the engine is typically long-lived, this is acceptable. `Close()` is available for explicit cleanup.
- `defer llmCancel()` ensures cancel is called even when `GenerateNextStep` panics or hangs, so the derived context is always cleaned up.

## Key decisions
- Added `Close()` method rather than making the goroutine self-cleaning via a timeout — the goroutine's lifetime is tied to the engine's, so explicit cleanup is more appropriate.
- `lifecycleCancel` stored as a field instead of using a finalizer — simpler and more idiomatic Go.
