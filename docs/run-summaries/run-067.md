## What happened

### Phase 1: TUI State Sync Fixes (4 bugs)

**Bug 1 — New sessions invisible after creation:**
- Root cause: `startCampaign` creates session in goroutine but never notifies TUI. TUI ticker picks up session in `m.sessions` but `m.currentRun` stays nil/old.
- Fix: Added `runCreatedMsg` + buffered channel bridge. `startCampaign` sends runID to channel, TUI receives via `runCreatedCmd`, auto-selects new session and triggers `refreshAllCmd`.
- Files: `commands.go`, `main.go`, `cmd/main.go`

**Bug 2 — Steering commands don't trigger refresh:**
- Root cause: `processSteeringCommand()` calls handlers (mutates store) but returns void. TUI waits 2s for ticker.
- Fix: Changed return type to `tea.Cmd`. Returns `fetchRunCmd` after successful retry, skip, pause, resume, approve, continue.
- Files: `main.go`

**Bug 3 — Spacebar pause/resume doesn't trigger refresh:**
- Same pattern. Returns `fetchRunCmd` after successful pause/resume/acknowledge.
- Files: `main.go`

**Bug 4 — Cancel key doesn't trigger refresh:**
- Same pattern. Returns `fetchRunCmd` after successful cancel.
- Files: `main.go`

## What to watch for
- Steering instructions cap at 5 (FIFO eviction) — older instructions are dropped.
- Instructions only checked between steps, not mid-tool-execution.
- `runCreatedCh` is buffered (size 1) — if TUI hasn't started listening, message is dropped (acceptable since ticker will pick up session anyway).
- `LifecycleController` is shared between TUI and engine — both read/write via thread-safe channels and mutexes.

## Key decisions
- Used channel-based notification for new sessions rather than polling — cleaner Bubbletea pattern.
- Steering instructions appended to LLM prompt with `IMPORTANT` label for visibility.
- Cap of 5 instructions prevents prompt bloat while keeping recent context.
- `SteeringEvent.Instruction` field added (not a separate type) to keep event model simple.
