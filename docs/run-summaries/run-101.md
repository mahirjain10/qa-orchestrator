## Goal
Fix 5 bugs across config.go, provider_openrouter.go, and client.go (LLM package).

## What happened

### Bug 1: parseTimeout useless wrapper (`config.go:185-187`)
- **Bug:** `parseTimeout` was a one-line delegate to `parseInt`.
- **Fix:** Removed the wrapper, call `parseInt` directly.

### Bug 2: No upper bound on MaxRetries (`config.go:201-203`)
- **Bug:** MaxRetries only checked `< 0`, allowing values like 1,000,000.
- **Fix:** Added `maxMaxRetries = 20` constant. Checks upper bound in both `LoadConfig` and `Validate`.

### Bug 3: ApplyProviderSettings overwrites existing (`provider_openrouter.go:100-124`)
- **Bug:** `ApplyProviderSettings` unconditionally did `p.Provider = &ProviderSettings{}`, discarding existing values set on the struct.
- **Fix:** Changed to `if p.Provider == nil { p.Provider = &ProviderSettings{} }` — only creates if nil, merges into existing otherwise.

### Bug 4: LLM_PROVIDER_ALLOW confusing name (`provider_openrouter.go:121-124`)
- **Bug:** Env var name `LLM_PROVIDER_ALLOW` implied a provider whitelist, but actually controlled `AllowFallbacks`.
- **Fix:** Renamed to `LLM_ALLOW_FALLBACKS`. Updated config field from `ProviderAllow` to `AllowFallbacks`. Updated `client_test.go`, `Makefile`, `.env.example`.

### Bug 5: Body read error non-retryable (`client.go:200-203`)
- **Bug:** `io.ReadAll(resp.Body)` error used `fmt.Errorf`, making I/O errors non-retryable.
- **Fix:** Wrapped with `NewRetryableError(..., true)` so transient I/O errors trigger retries.
