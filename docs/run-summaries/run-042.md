## What to watch for
- **No mutexes needed**: Bubble Tea's update loop runs single-threaded. All state mutations happen in `Update()`, which is called sequentially by the framework.
- **Async commands return `tea.Msg`**: The command function returns a closure that runs in a goroutine and sends a message back to the update loop. This is the standard Bubble Tea pattern.
- **`refreshAllCmd` batches 5 fetches**: All data fetches run in parallel via `tea.Batch()`. This is faster than sequential but means all 5 I/O operations happen on every tick.
- **`startRefreshTicker` returns nil for empty runID**: When no run is selected, the ticker stops. This prevents unnecessary I/O when idle.
- **`updateFromStores()` is test-only**: This synchronous method exists solely for testing. Production code uses async commands.
- **`errMsg` uses Go's `error` interface**: Any error type works ‚Äî tests use a custom `testError` struct.
