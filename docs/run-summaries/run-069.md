## Goal
Implement `observe_ui()` tool to provide grounded page observations to the autonomous planner, eliminating selector hallucination in autonomous flows.

## What happened

### Phase 1: JS Extraction Script + Tool Registration
**File:** `packages/browser-runtime/tools/registry.go`

- Added inline JS extraction script constant (`observeUIJS`) using `document.querySelectorAll()`.
- Extracts visible interactive elements: input, button, textarea, select, a, [role="button"], [role="textbox"].
- Filters: hidden elements, zero-size elements, opacity=0.
- Generates selectors using priority: `#id` → `tag[name]` → `text selector` → `nth-of-type`.
- Caps at 50 elements, sorted by priority (inputs → buttons → forms → links).
- Returns JSON: `{"page_state": "loaded"|"empty", "interactive": [...]}`.
- Registered `observe_ui` tool in `registerDefaultTools()` using `page.Evaluate()`.

### Phase 2: Mock observe_ui
**File:** `packages/agents/executor/executor.go`

- Added mock `observe_ui` to `MockToolRegistry` returning `{"page_state": "loaded", "interactive": []}`.
- Ensures tests pass without Playwright.

### Phase 3: Auto-Observation After Navigate
**File:** `packages/agents/engine/engine.go`

- Added `autoObserve()` method to `AgentEngine`.
- In `executeAndValidate()`, after successful `navigate` step, calls `autoObserve()`.
- Appends observation to `ctx.Observations` slice — appears as "Current Observation" in LLM prompt.
- Uses `HasTool()` interface check (with fallback for mock registry).

### Phase 4: Re-Observation on Failure
**File:** `packages/agents/engine/engine.go`

- In `executeAndValidate()`, on step failure, calls `autoObserve()` before returning.
- Recovery handler sees fresh page state for decision making.
- Handles: click failures, wait timeouts, selector missing, stale elements.

### Phase 5: System Prompt Update
**File:** `packages/llm/prompts.go`

- Added `## Selector Rules (CRITICAL)` section to `SystemPromptTemplate`:
  - "Use selectors from Current Observation whenever possible"
  - "Do not invent selectors if observed elements exist"
  - "If Current Observation contains interactive elements, use their exact selectors"
  - "After navigating to a page, the observation will contain discovered elements with their selectors"

## What to watch for
- 300ms delay after navigate is initial; may need increase for SPAs.
- 50 element cap keeps token cost manageable for GPT-4o.
- Empty pages return `{"page_state": "empty", "interactive": []}` — no error.
- `observe_ui()` is LLM-callable too, not just auto-triggered.

## Key decisions
- Inline JS string (not separate .js file) — POC-appropriate, single file.
- Auto-trigger by engine (not LLM prompt rule) — deterministic, reliable.
- Appends to Observations slice (not dedicated field) — fits existing prompt flow.
- Re-observation BEFORE recovery handling — recovery agent sees fresh state.
- Selector generation: simple priority chain, no advanced scoring.
