## What happened

### Fix 1: Windows TempDir Cleanup Race (`apps/tui/cmd/main_test.go`)
**Root cause:** `t.TempDir()` cleanup fails on Windows with `unlinkat ... \traces: The directory is not empty` because `os.Rename` (used by all three stores' `persist` methods) can leave file handles briefly open after write completes. When `t.TempDir()` runs its deferred `RemoveAll`, the directory is still locked.

**Fix:** Replaced `t.TempDir()` with `os.MkdirTemp()` + custom `t.Cleanup` with retry logic (5 attempts, 100ms delay between retries). Added `makeTestDir()` helper function. Removed `os.RemoveAll(dataDir)` from `createStores()` to avoid double-cleanup conflict.

**Files changed:**
- `apps/tui/cmd/main_test.go` — replaced 3 `t.TempDir()` calls with `makeTestDir()`, added retry cleanup logic, removed redundant cleanup from `createStores()`

### Fix 2: Missing Cancelled Status Color (`components/run_panel.go:60,133`)
**Bug:** `ViewWithWidth` method's status color switch was missing `RunStateCancelled` and `RunStateCancelling` cases. Cancelled runs showed as gray "Pending" instead of dim "Cancelled".

**Fix:** Added `case types.RunStateCancelled, types.RunStateCancelling: statusColor = style.StatusCancelled` to both `View()` and `ViewWithWidth()` methods.

### Fix 3: Slice Panic on Short Artifact IDs (`components/trace_panel.go:207`)
**Bug:** `a.ArtifactID[:12]` panics on short IDs (tests use `"art-1"` = 5 chars, but production uses `"art_" + 16 hex chars = 20 chars).

**Fix:** Added length guard: `if len(idStr) > 12 { idStr = idStr[:12] }`
