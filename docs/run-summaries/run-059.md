## What happened

### Problem
- Traces, flows, and reports did not update automatically during campaign execution
- User had to press 'r' to manually refresh
- Root cause: startRefreshTicker("") returned nil when no run was selected at startup, killing the ticker permanently

### Bug Chain
1. At startup, m.currentRun is nil, currentRunID() returns ""
2. First tickMsg fires after 2 seconds
3. startRefreshTicker("") returns nil — ticker dies
4. No more ticks ever fire, even when run is later selected

### Implementation
1. Fixed startRefreshTicker() in commands.go:
   - Removed runID parameter — always returns tea.Cmd, never nil
   - Ticker is now self-perpetuating

2. Updated Init() in main.go:
   - Calls startRefreshTicker() without runID

3. Updated tickMsg handler:
   - Always re-schedules ticker via startRefreshTicker()
   - Fetch functions already handle empty runID gracefully

4. Added fetch-on-select trigger:
   - When campaign is selected via Enter key, immediately triggers refreshAllCmd and startRefreshTicker

5. Updated existing tests in commands_test.go:
   - Replaced TestStartRefreshTickerReturnsNilForEmptyRunID and TestStartRefreshTickerReturnsCommandForValidRunID with single TestStartRefreshTickerAlwaysReturnsCommand
