## What happened

### O1: Flow timeout validation allows 0 (parser.go:110-112)
- **File:** `packages/orchestrator/campaign/parser.go`
- **Bug:** Flow timeout check was `!= 0 && < 0` — allowed 0, while campaign-level check used `<= 0`. Inconsistent.
- **Fix:** Changed flow check to `< 0` (0 means "inherit from campaign timeout"). Added hint in error message: `(0 = inherit from campaign)`.

### O2: topologicalSort returns partial order without check (dependencies.go:110-149)
- **File:** `packages/orchestrator/validator/dependencies.go`
- **Bug:** `Validate()` called `topologicalSort()` and unconditionally returned `Valid: true`. If the sort missed flows (cycle not caught by DFS), validation would still pass.
- **Fix:** Added `len(order) < len(flows)` check in `Validate()` after `topologicalSort()`. If flows are missing, constructs the cycle deps list and returns `Valid: false` with a `DependencyError`.

### O3: FormatError panics on nil *DependencyError (dependencies.go:173-189)
- **File:** `packages/orchestrator/validator/dependencies.go`
- **Bug:** `FormatError(err *DependencyError)` accessed `err.MissingDeps` without nil guard.
- **Fix:** Added `if err == nil { return "unknown dependency error" }` guard at function start.

### O4: DependencyError.Error() returns generic messages (campaign.go:143-151)
- **File:** `packages/shared/types/campaign.go`
- **Bug:** `Error()` returned `"missing dependencies"` / `"circular dependency detected"` with no flow ID or details.
- **Fix:** Changed to `flow "f1": missing dependencies: missing1` and `flow "f1": circular dependency detected in: a -> b`. Added `fmt` and `strings` imports.

### O5: No config-block-existence check in parser (parser.go:86-94)
- **File:** `packages/orchestrator/campaign/parser.go`
- **Bug:** When config block was omitted, field-level errors fired (e.g., `'timeout' must be greater than 0`) instead of a clear "config block required" message.
- **Fix:** Added `campaign.Config == (CampaignConfig{})` check before field-level checks, producing a clear error: `campaign 'config' block is required (must specify 'timeout', 'retry_limit', and 'parallel_limit')`.
