## Goal
Fix TUI bugs T4 and T5 from bug-report-sunday.md: ANSI-column alignment in Flows view and context leak on campaign startup error.

## What happened
1. **T4 — ANSI escape codes misalign table columns (`apps/tui/internal/screens/main.go:882-886`):** `fmt.Sprintf` width specifiers (`%-*s`) count ANSI escape sequences from `lipgloss.Style.Render()` as visible characters, causing the status column to be incorrectly padded — shorter than other columns when styled. Fixed by computing the visible width with `lipgloss.Width(styledStr)` and manually right-padding the styled status string, bypassing Sprintf width specifiers for that column.

2. **T5 — Context not cancelled on campaign startup error (`apps/tui/cmd/main.go:101-106`):** When `startCampaign` returned an error, the goroutine only logged the error but never called `campaignCancel()`. The context remained active, leaking any goroutines spawned with `campaignCtx`. Fixed by calling `campaignCancel()` before the error log.
