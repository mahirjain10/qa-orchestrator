## Goal
Fix bugs S8, S9, S11, S12 from bug-report-sunday.md across the browser-runtime package.

## What happened
1. **S8 — Page recovery in non-Navigate ops (`runtime.go`):** `FlowBrowserRuntime` already used `withPageRecreation` for all ops, but `BrowserRuntime` (parent) only had inline page-close recovery in `Navigate`. Added a shared `withPageRecreation` helper to `BrowserRuntime` and refactored all 7 methods (`Navigate`, `Click`, `Fill`, `WaitForSelector`, `TextContent`, `InnerHTML`, `Evaluate`, `Screenshot`) to use it. ~60 lines of duplicate lock/ensurePage boilerplate removed.

2. **S9 — Stop() doesn't nil out fields (`runtime.go:133-162`):** `BrowserRuntime.Stop()` and `FlowBrowserRuntime.Close()` closed page/context/browser/playwright but left stale references in the struct fields. After `Stop()`, `r.page`, `r.context`, `r.browser`, and `r.pw` now set to `nil`.

3. **S11 — observe_ui ignores Evaluate errors (`tools/registry.go:372,388`):** `headingResult, _` and `titleResult, _` silently discarded errors from `runtime.Evaluate`. Changed to capture errors — if Evaluate fails, the heading/title 404 check is skipped gracefully instead of silently failing.

4. **S12 — matchesType no json.Number (`tools/registry.go:549`):** `matchesType("number", value)` returned `false` when `value` was a `json.Number` (which occurs when parsing LLM tool-call responses). Added `json.Number` to the type switch.
