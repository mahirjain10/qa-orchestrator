Fix Phase 7 blockers: autonomous one-step termination bug and guided-mode replan misuse.

## What happened
1. Autonomous loop termination fixed:
   - Updated planner `ShouldStop()` to never terminate autonomous plans based only on consumed generated steps.
   - This removes the one-and-done behavior where autonomous execution stopped after step 1.
2. Finish signal implemented:
   - Added explicit `finish` tool signal handling in `runAutonomousFlow`.
   - When LLM returns `tool=finish`, engine exits loop cleanly without forcing execution to max-step limit.
3. Guided recovery hardening:
   - In guided mode, `RecoveryActionReplan` now falls back to retry behavior instead of rebuilding the same static YAML plan.
   - Prevents useless guided replanning loops.
4. Regression tests added:
   - `TestAutonomousFlow_MultiTurnUntilFinish` verifies multi-turn generation and finish termination.
   - `TestGuidedFlow_ReplanFallsBackToRetry` verifies guided locator-style recovery behavior.
   - `TestPlannerShouldStop_AutonomousWithConsumedSteps` locks autonomous ShouldStop semantics.
