## What happened

### Problem
- All sessions appeared identical in campaign selector — no way to distinguish current run from historical runs
- Session.StartedAt field existed but was never used for rendering

### Implementation
1. Added formatSessionAge() helper:
   - Returns "just now" for < 1 minute
   - Returns "Xm ago" for < 1 hour
   - Returns "Xh ago" for < 24 hours
   - Returns "Xd ago" for >= 24 hours
   - Returns "unknown" for zero time

2. Updated renderCampaignSelector():
   - Splits sessions into currentSession and previousSessions
   - Current session rendered first with bright green "●" indicator
   - Previous sessions sorted by StartedAt descending (newest first)
   - Separator line between current and previous sessions
   - Each session shows age in brackets: [5m ago]

3. Updated campaignNames():
   - Includes age label: "Campaign [run-id] (5m ago)"
   - Adds [CURRENT] marker for active run
