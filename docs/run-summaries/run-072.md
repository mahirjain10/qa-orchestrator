# Run 072 Summary

**Date:** 2026-05-21
**Agent:** qwen3.6-plus-free
**Scope:** Autonomous Planner Selector Hallucination Fix

## Problem
Across multiple runs, 3/8 flows consistently failed with "max retries (3) exceeded" due to 30-second timeouts on selectors that don't exist on the page:
- flow-epsilon: `wait_for(input[type='file'])` → never appears → FAILED
- flow-beta: `wait_for(table)` → never appears → FAILED
- flow-gamma: `get_text(h1)` → never appears → FAILED

Root cause: LLM generates steps with invented selectors before having observation data. Prompt-only enforcement (RULE 1-5, decision tree checklist) was completely ignored by gpt-4o — zero `observe_ui()` calls across all flows.

## Solution: 4-Layer Defense

### Layer 1: Code-Level Enforce observe_ui (engine.go)
- `injectObserveStep()` auto-injects `observe_ui` as a REAL step after every successful `navigate()`
- Also injects `observe_ui` before every retry/replan after failure
- Makes observe_ui visible in plan history, not a hidden side-effect
- Observation data flows through `formatObserveUIObservation()` to planner prompt

### Layer 2: Recovery Agent Fix (recovery.go)
- Added `isSelectorTimeout()` — detects Playwright selector timeouts (`playwright` + `timeout`)
- Selector timeouts now trigger `Replan` instead of blind `Retry`
- Removed "timeout" from `isNetworkError()` to prevent false matches
- Added "not found on page" and "does not exist" to `isLocatorError()`
- Before: failure → retry same selector 3x → die. After: failure → replan with fresh observation

### Layer 3: Pre-Execution Selector Check (registry.go)
- Added `selectorExistsJS` — fast JS check using `document.querySelector()`
- Added `checkSelectorExists()` to ToolRegistry
- Modified `click`, `wait_for`, `get_text`, `get_html`, `type_text` to run fast check first
- Turns 30-second timeouts into sub-second failures for nonexistent selectors
- Recovery agent recognizes the new error message and triggers replan

### Layer 4: User Prompt Reformatted (prompts.go)
- Moved `{{.Observation}}` to very bottom of user prompt (attention recency bias)
- Added aggressive header: `## USE ONLY THESE SELECTORS` with explicit instructions
- Observation now appears right before the output instruction

### Trace Visibility (engine.go)
- Added `buildObservationSummary()` helper
- Added trace event `observation_context` before each planner call
- Logs: last observation tool, success, page state, element count
