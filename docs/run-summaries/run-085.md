## Goal
Fix 3 root-cause bugs identified in the real-urls-parallel trace analysis: false passes after loop detection, URL alternation loops, and :has-text() selector failures.

## What happened

### 1. observeUIJS DOM tree walk rewrite
Replaced the hardcoded `document.querySelectorAll('input, button, textarea, select, a, h1, h2... .title, .shopping_cart_link...')` with `document.createTreeWalker` that walks the full DOM. The new JS:
- Uses `isVisible()` + `isCapturable()` runtime checks to include any visible element with an ARIA role, meaningful text, or interactive tag
- Generates selectors in priority: `#id` → `tag[name]` → `[data-test="..."]` → `tag.class` → `tag:nth-of-type(N)`
- No more `:has-text()` — every selector works with `document.querySelector()`
- No more per-site patches (`.title`, `.shopping_cart_link`, etc.)
- Same output format (tag, text, id, name, class, placeholder, selector)
- Uses `CSS.escape()` for safe class-name selectors

### 2. Loop steering text fix (engine.go:420, 436)
Changed the loop detection steering instructions from "Try a different approach **or use finish**" to "Try a different approach. **Do NOT finish with success** unless observations confirm the goal is met." This prevents the LLM from taking the "escape hatch" of hallucinating success after a loop detection.

### 3. Block finish(success) after loop detection (engine.go:382-407)
Added a new safety-net check: if the LLM tries `finish(success)` when `consecutiveRepeats > 0`, the call is blocked and a steering instruction is injected telling the LLM to either try a different approach or use `finish(fail)`. This prevents the false-pass pattern where loop detection → LLM hallucinates "I see the heading" → finishes with success.

### 4. Alternation detection (engine.go + agent.go)
Added `VisitedURLs map[string]bool` to `ExecutionContext`. Before executing a `navigate` step, the engine checks if the URL is already in the visited set. If so, it injects a steering instruction and skips the step. After a successful navigation, the URL is added to the visited set. This prevents A→B→A→B alternation loops without relying on step-signature matching (which can't detect alternating patterns). Checkpoint save/restore also handles `VisitedURLs`.

## What to watch for
- `VisitedURLs` is per-flow, so each independent flow starts fresh. This is correct for parallel execution.
- `CSS.escape()` is available in all modern Playwright browsers (Chromium/Firefox/WebKit).
- The alternation check uses `ctx.VisitedURLs[url]` lookup — if the map is nil, it's a no-op (handled by nil check).
- `:has-text()` selectors are completely gone from observeUIJS output. If any existing code depends on them, `extractTextFromSelector()` still parses them (for the auto-replace safety net when the LLM hallucinates old-format selectors).

## Key decisions
- Added `VisitedURLs` to `ExecutionContext` rather than using a local variable in `runAutonomousFlow()` so checkpoint save/restore preserves the visited set across pause/resume.
- Block `finish(success)` by checking `consecutiveRepeats > 0` — this works because the finish handler runs BEFORE the repeat-detection reset, so the counter is still positive.
- Used `CSS.escape()` for class name selectors rather than a manual regex escape — safer and simpler.
