## What happened

### Phase 12 Implementation:

**Added hard minimum terminal size checks (main.go:409-420)**
- Replaced combined `width < 80 || height < 24` check with separate checks
- Width < 80: `style.StatusFailed.Render("Terminal too narrow (min 80 columns). Current: <width>")`
- Height < 20: `style.StatusFailed.Render("Terminal too short (min 20 rows). Current: <height>")`
- Uses styled error messages instead of dim text

**Added `sidebarWidth()` helper (main.go:766-774)**
- Returns 24 at width >= 100
- Returns 20 at width >= 90 and < 100
- Returns 16 at width < 90
- Collapses sidebar on narrow terminals

**Refactored `contentWidth()` (main.go:776-782)**
- Now uses `m.sidebarWidth()` instead of hardcoded 26
- Formula: `m.width - m.sidebarWidth() - 4`
- Maintains minimum of 40

**Added `contentHeight()` helper (main.go:784-789)**
- Returns `m.height - 3` when height < 25 (hides status bar hints)
- Returns `m.height - 5` when height >= 25 (full layout)

**Updated `View()` (main.go:409-445)**
- Uses `m.sidebarWidth()` for sidebar sizing
- Uses `m.contentHeight()` for body height
- Content width calculated as `m.width - sidebarWidth - 2`

**Updated `renderTracesView()` (main.go:791-799)**
- Now uses `m.contentHeight() - 2` instead of `m.height - 7`
- Consistent with adaptive height calculation
