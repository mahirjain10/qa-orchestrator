## What happened

### Phase A ‚Äî Critical Bug Fixes
1. **JS injection in `checkSelectorExists`** (registry.go:120): Changed string concatenation `selectorExistsJS + (" + selector + ")` to use `json.Marshal(selector)` which properly escapes quotes, backticks, and parentheses. An attacker-controlled selector like `"); alert(1)//` no longer escapes the JS string context.
2. **Fragile "cancelled" string check** (engine.go:1130): Changed `strings.Contains(result.Errors[0], "cancelled")` to `strings.HasPrefix(result.Errors[0], cancelledErrPrefix)` using a sentinel prefix constant. Prevents false-positives from error messages that happen to contain "cancelled" as a substring.

### Phase B ‚Äî Resume-From-Pause Fix
3. **Extended checkpoint** (engine.go:818-824): Now saves `CurrentURL`, `LastStepSignature`, and `ConsecutiveObserveCount` in checkpoint payload alongside existing step fields.
4. **Restore on resume** (engine.go): Added `restoreCheckpoint()` method that reads the saved checkpoint from session store and restores `CurrentURL`, `LastStepSignature`, and `ConsecutiveObserveCount` into the execution context. Called after every `waitForResume()` in both guided and autonomous loops.

### Phase C ‚Äî Guided Flow Steering + Minor Fixes
5. **Steering drain in guided loop** (engine.go): Added `DrainSteeringEvents()` call in `runGuidedFlow` after pause state checks and before each step execution. Previously, steering instructions (retry, skip, pause text) during guided execution were never processed because only the autonomous loop drained events.
6. **Include planStep.Reason in finish(fail)** (engine.go:460): The LLM's explanation (`planStep.Reason`) is now included in `result.Errors` instead of only being logged in the trace event. Error message format: `"LLM signaled that the goal is unachievable: <reason>"`.
7. **Documented RecoveryActionReplan** (agent.go:132, engine.go:560): Added clarifying comments noting that `RecoveryActionReplan` is semantically retry-with-observation, not a true replan.

## What to watch for
