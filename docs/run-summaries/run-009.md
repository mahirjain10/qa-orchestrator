## What went wrong
- The `ToolRegistry` used an unprotected map `map[string]Tool` for tool storage and retrieval. In a multi-agent orchestrated environment, this is highly prone to race conditions and panics.

## What to watch for
- Always protect shared global state (like a registry map) with a `sync.RWMutex` if it will be read and written concurrently by multiple goroutines.
- Use `go test -race` locally to actively hunt for concurrency flaws before calling a phase complete.

## Key decisions
- Implemented robust `RWMutex` locking around all registry operations (`Register`, `Execute`, `HasTool`, `ListTools`) to guarantee thread safety.
