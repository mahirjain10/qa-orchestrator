## Goal
Fix the root cause of selector hallucination in autonomous flows: the planner always saw the validator observation instead of the observe_ui observation because `CreateObservation` was appended AFTER `autoObserve`. Additionally, improve the planner's observation formatting to expose structured observe_ui data (interactive elements with selectors) to the LLM.

## What happened

### Bug Analysis
Traced through the actual execution log from the previous run to confirm the root cause:

**Before fix (engine.go:483-508):**
```
Successful navigate flow:
  1. ExecuteStep → success
  2. autoObserve() → appends observe_ui at index N
  3. CreateObservation → appends navigate obs at index N+1  ← becomes "last"
  4. Planner reads Observations[len-1] → sees navigate, NOT observe_ui ❌

Failed step flow:
  1. ExecuteStep → fails
  2. CreateObservation → appends validator obs at index N
  3. autoObserve() → appends observe_ui at index N+1  ← becomes "last"
  4. Planner reads Observations[len-1] → sees observe_ui ✓
```

The planner at `planner.go:122` reads `Observations[len-1]`, so on successful navigate steps, the LLM had zero visibility into what observe_ui found on the page.

### Phase 1: Fix Observation Ordering
**File:** `packages/agents/engine/engine.go`

- Removed conditional `autoObserve()` calls (navigate-success only, failure only).
- Replaced with single unconditional `autoObserve()` call AFTER `CreateObservation` append and validation.
- This ensures observe_ui is ALWAYS the last observation regardless of step success/failure.
- Changed `autoObserve()` signature to accept `stepResult` for trace context.

### Phase 2: Improve Planner Observation Formatting
**File:** `packages/agents/planner/planner.go`

- Added `formatObserveUIObservation()` helper function.
- Detects when last observation is from `observe_ui` and formats data prominently:
  - Page state (loaded/empty)
  - Interactive element count
  - Each element: tag, selector, text content
  - Warning: "Use the selectors above. Do not invent selectors."
- Falls back to raw data display if JSON parsing fails.
- Non-observe_ui observations use the original formatting.

### Phase 3: Add Trace Emission
**File:** `packages/agents/engine/engine.go`

- Added `trace.EmitAgentDecision()` call inside `autoObserve()`.
- Emits: agent="engine", action="auto_observe", reason="triggered after {tool} (success={bool})".
- Makes auto-observation visible in trace logs for debugging.

## What to watch for
- `autoObserve()` is now called unconditionally after every step (not just navigate/failure). This means more observe_ui calls — monitor token cost.
- The `formatObserveUIObservation` function caps at displaying all interactive elements. If observe_ui returns 50 elements, all 50 are formatted. Consider truncation if token cost is high.
- Trace emission uses `EmitAgentDecision` with agent="engine" — may want a dedicated `EmitAutoObserve` event type in the future.

## Key decisions
- Unconditional `autoObserve()` instead of navigate-only — every step can change page state (click, wait, etc.), so observe_ui after every step gives the planner the most accurate view.
- Structured formatting over raw JSON — LLM consumes formatted element list much more reliably than raw JSON in a single line.
- Backward compatible — non-observe_ui observations use original formatting, existing tests unchanged.
