Implement V4 Debug Fix Phase 1 — Fix Keyboard Interactivity & Global Key Routing.

## What happened

### Problem
- Keyboard shortcuts (help, quit, view switch) were blocked when in steering or filter mode
- '?' was referenced in contextualKeys() but had no handler
- Users were trapped in steering/filter mode with only ESC as escape

### Root Cause
- handleFilterKey and handleSteeringKey swallowed ALL keys, blocking global keys
- handleMainKey had no case for '?'
- No help modal existed

### Implementation
1. Added showHelp bool field to MainScreen struct
2. Added global key pass-through in Update() BEFORE mode checks:
   - q / ctrl+c → tea.Quit (always works)
   - ? → toggles showHelp (always works)
3. Implemented renderHelpModal() with contextual keys per view:
   - Global section (always shown): q/ctrl+c quit, ? toggle help
   - Dashboard: 1-4 switch views, TAB focus, up/down navigate, enter select, space pause, x cancel, s steer, r refresh
   - Flows: enter expand/collapse, left/h collapse
   - Traces: / filter, S failures-only, F/f follow-tail, pgup/pgdown scroll
   - Report: r refresh
   - Steering: enter execute, esc exit
   - Filter: enter apply, esc cancel
4. Updated View() to render help modal overlay when showHelp is true
5. Updated handleSteeringKey and handleFilterKey to allow q/ctrl+c to pass through
