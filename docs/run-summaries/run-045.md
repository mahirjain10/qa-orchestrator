## What happened

### Fix #1: Discarded RunFlow Result (`cmd/main.go:162-170`)
**Bug:** `runCampaign` discarded `eng.RunFlow()` result with `_ = result`. Flow-level errors were silently lost — no session state update on pass/fail.

**Fix:** Now checks `result.Outcome` and updates flow state accordingly:
- `OutcomeFail` → `UpdateFlowState(..., FlowStateFailed, result.Errors[0])`
- `OutcomePass` → `UpdateFlowState(..., FlowStatePassed, "")`
- `OutcomeSkip` → no action (handled elsewhere)

### Fix #2: Side Effect in View() (`screens/main.go:491`)
**Bug:** `renderComponent` called `m.campaignList.SetCampaigns(m.campaignNames())` inside the View render path. This violates Elm architecture — View() should be pure, with no state mutations.

**Fix:** Removed the `SetCampaigns()` call from `renderComponent`. The campaign list is already populated by:
- `sessionsLoadedMsg` handler in `Update()` (async path)
- `updateFromStores()` (test/sync path)

### Fix #3: Space Bar Pause/Resume Gap (`screens/main.go:327-344`)
**Bug:** After `PauseRun` transitions to `Pausing`, pressing space again did nothing. `ResumeRun` only accepts `Paused` state, so the intermediate `Pausing` state was a dead zone. Same issue for `Resuming` and `Cancelling`.

**Fix:** Updated space bar handler to handle all intermediate lifecycle states:
- `Pausing` → shows "Run is pausing, please wait"
- `Resuming` → shows "Run is resuming, please wait"
- `Cancelling` → shows "Run is cancelling, please wait"
- `Pending`/`Running` → pause (message: "Run pausing...")
- `Paused` → resume (message: "Run resuming...")

### Fix #4: Custom `itoa` Instead of `strconv` (`util/truncate.go:76-85`)
**Bug:** Custom `itoa` function re-implemented integer-to-string conversion instead of using `strconv.Itoa`. Unnecessary code, potential for subtle bugs.

**Fix:** Replaced custom `itoa` with `strconv.Itoa`. Removed the 10-line `itoa` function entirely. Added `strconv` import.
