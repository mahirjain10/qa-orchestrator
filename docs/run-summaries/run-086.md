## Goal
Analyze 7 trace log files to identify root causes of flow failures and implement fixes.

## What happened

### Trace Analysis
Read all 7 trace files in `data/traces/` (excluding old/ and v1/). Identified 5 root cause issues:

| Priority | Issue | Impact |
|----------|-------|--------|
| **P0** | `Plan.GetHistory()` omits step success/failure status | LLM re-generates already-executed steps, burning 2-3x more calls |
| **P1** | `performRootNav()` doesn't reset `VisitedURLs` | URL alternation deadlock after root nav — LLM can't re-navigate to target |
| **P2** | No `echo` tool in real browser `ToolRegistry` | All 6 guided-parallel-test flows fail instantly |
| **P2** | `blocked_finish_success` has no auto-fail | LLM loops indefinitely on blocked finish attempts |
| **P3** | HITL wait state never pauses | Flow burns through max steps without waiting |

### Fixes Implemented

#### P0: Planner loop bug (agent.go + engine.go)
- Added `Result *StepResult` field to `PlanStep` struct
- Updated `GetHistory()` to render `[SUCCESS]` or `[FAILED: error]` for executed steps
- Stored step result back into plan step after `executeAndValidate()` in both guided and autonomous flows
- **Effect:** LLM sees actual outcome of each step and won't re-generate completed steps

#### P1: URL alternation deadlock (engine.go)
- Reset `ctx.VisitedURLs` to `{rootDomain: true}` in `performRootNav()`
- **Effect:** After 404 recovery navigates to root domain, LLM can navigate to target URLs again

#### P2.1: Missing echo tool (registry.go)
- Added `registerEcho()` method to browser-runtime `ToolRegistry`
- Registered in `registerDefaultTools()` line
- Added echo description to `getToolDescription()`
- **Effect:** All 6 guided-parallel-test flows now have the echo tool available

#### P2.2: Blocked finish success auto-fail (engine.go)
- Added `blockedFinishSuccessCount` local counter in autonomous flow loop
- If LLM tries `finish(success)` 3+ consecutive times after loop detection, auto-fails with `OutcomeFail`
- **Effect:** Prevents infinite blocked-finish loops

## What to watch for

- `GetHistory()` renders `[FAILED: unknown error]` when `Result.Error` is nil but `Result.Success == false`

## Key decisions
- Stored `Result` on `PlanStep` rather than adding a separate step-result map — simpler, no lookup needed
- Used local `blockedFinishSuccessCount` counter rather than storing in `ExecutionContext` — no need for checkpoint persistence since this is a per-loop bounded counter
- `VisitedURLs` reset to `{rootDomain: true}` rather than full clear — prevents re-navigation to the root domain itself (which would be pointless)
