## Goal
Fix 5 logic errors identified in the codebase: (7) Playwright resource leak on context re-creation. (8) Invalid selector syntax silently masked as "not found". (9) State corruption on campaign resume with parallel flows. (10) Silent error swallowing on HTTP failure. (11) Orphaned artifact files on disk after failed deletion.

## What happened

### Bug 7: Playwright Resource Leak on Context Re-creation
- `packages/browser-runtime/runtime.go` — `ensurePage()` method.
- When `r.context.NewPage()` fails, the code fell through to creating a new context via `r.browser.NewContext()` and overwrote `r.context` without calling `.Close()` on the old one, leaking Playwright browser processes.
- Fixed by adding `r.context.Close()` before overwriting with the new context.

### Bug 8: Invalid Selectors Silently Masked as "Not Found"
- `packages/browser-runtime/tools/registry.go` — `checkSelectorExists()` method.
- The JS `selectorExistsJS` returns `{"exists": false, "error": "invalid selector syntax"}` on bad CSS. The Go code ignored the `error` field and always reported "not found on page".
- Fixed by checking `check.Error` first and returning it as `"invalid selector syntax for '%s': %s"`.

### Bug 9: State Corruption on Campaign Resume
- `apps/tui/cmd/main.go` — `startCampaign()` and `runCampaignWithContext()`.
- The resume logic found the first incomplete flow and sliced `camp.Flows = camp.Flows[startIndex:]`. If a parallel flow after that index had already completed, the slice dropped it from `flowMap`, forcing duplicate execution on resume.
- Fixed with two changes:
  1. In `startCampaign()`: removed the slicing line. Now all flows stay in `camp.Flows`.
  2. In `runCampaignWithContext()`: pre-seeds `flowResults` from session state for completed flows, and skips them in the topoOrder loop (closes their `flowDone` channel).

### Bug 10: Silent Error Swallowing on HTTP Failures
- `packages/llm/client.go` — `doRequest()` method.
- If `io.ReadAll(resp.Body)` failed on a non-200 response, the returned error only contained the status code. The body read error was silently discarded.
- Fixed by wrapping `err` into the error message with `fmt.Errorf("reading error response body: %w (status %d, ...)")`.

### Bug 11: Orphaned Artifact Files on Disk
- `packages/storage/artifact/store.go` — `Delete()` method.
- If `os.Remove(a.Path)` failed (e.g. file in use on Windows), the store still called `delete(s.index, a.ArtifactID)` on line 203. The physical file remained on disk with no reference.
- Fixed by adding `continue` after the error block, so the index deletion only runs when file removal succeeds.

## What to watch for
- `ensurePage()` now calls `r.context.Close()` — if the context is nil, the nil check guards it.
- The `check.Error` field from `selectorExistsJS` now surfaces to the caller as a format error — any test that expects a "not found" message for invalid CSS may need updating.
- `startCampaign()` no longer modifies `camp.Flows` — all flows always remain in the list. The orchestrator skips completed ones via the `flowResults` pre-seed.
- The `Delete()` method in artifact store now `continue`s on `os.Remove` failure — the in-memory index stays consistent with disk state.

## Key decisions
- **Bug 9 approach:** Rather than modifying `runCampaignWithContext`'s goroutine to check session state at runtime, we pre-seed `flowResults` before the topoOrder loop. This keeps the goroutine logic clean and avoids adding a session read to every goroutine launch.
- The `continue` in the artifact `Delete()` loop ensures index consistency — the file either exists on disk and in index, or neither.
- All fixes are minimal (1-3 lines each) to reduce risk of regressions.
