## Goal
Fix three bugs discovered in SauceDemo E2E campaign run: observe_ui missing page title elements, broken upstream skip chain, and invisible cart icon selectors.

## What happened
1. **Bug diagnosis from trace** — Analyzed a real SauceDemo run trace showing login-flow failed despite successful login, cart-flow running without session, and confirmation-flow stuck on "not logged in" error page.

2. **Fix #1 — Upstream skip propagation (main.go:415)** — Changed `depResult.Outcome == engine.OutcomeFail` to `depResult.Outcome == engine.OutcomeSkip`. Previously, if login-flow failed (OutcomeFail) and browse-flow was skipped (OutcomeSkip), cart-flow didn't detect OutcomeSkip as a reason to stop and ran anyway without a session. Now only OutcomeSkip triggers upstream skip, while OutcomeFail (a flow that ran but failed) is allowed to pass its storage state downstream.

3. **Fix #2 — Widen observe_ui CSS selectors (registry.go:14)** — Added `.title, [data-test="title"], .shopping_cart_link, .app_logo, [class*="heading"]` to the querySelectorAll string. The SauceDemo "Products" page title is a `<span class="title">` — not an h1-h6 tag — so it was invisible to the LLM. Now the LLM can see it and verify the inventory page loaded.

4. **Fix #3 — Class field in observe_ui output (registry.go + planner.go)** — Added `class: el.className || ''` to the element data returned by the observe_ui JS, and added the `class` field to the LLM prompt format in `formatObserveUIObservation`. This lets the LLM see class names like `shopping_cart_link` instead of guessing from `a:nth-of-type(N)` selectors.

## What to watch for
- The `class` field in observe_ui output is new — the JS always returns it (may be empty string). The Go format code handles missing `class` gracefully.
- Adding more CSS selectors to observe_ui increases candidate count but should rarely hit the 50-element cap.
- `OutcomeFail` flows now pass storage state downstream — if this is undesirable for some failure scenarios, the skip logic may need per-flow configuration.

## Key decisions
- Changed skip trigger from OutcomeFail to OutcomeSkip — a failed flow ran and may have useful state (cookies, localStorage), while a skipped flow has nothing.
- Used specific CSS classes (`.title`, `.shopping_cart_link`) rather than generic tag selectors like `span` — avoids returning irrelevant elements.
- Added class to both JS output and LLM prompt format — minimal change (6 chars per element line) that gives the LLM critical identification hints.
