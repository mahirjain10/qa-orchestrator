## What went wrong
- playwright-go v1.52.0 doesn't exist - version format is v0.XXXX.X, not v1.XX.0
- Playwright API changes between versions - had to fix type conversions (int64 vs int, struct field names)
- Empty flow plan returned nil, causing nil pointer panic

## What to watch for
- playwright-go uses version format v0.XXXX.X (e.g., v0.5700.1, NOT v1.52.0)
- Always search GitHub for actual available versions before specifying
- Playwright API returns pointers (e.g., `*float64`) not values - use playwright.Bool(), playwright.Float()
- Browser types use `BrowserTypeChromium` not string literals
- Don't return nil for empty plans - return empty slice instead
- Empty flows with len(steps)==0 need special handling in planner
