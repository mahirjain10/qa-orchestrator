**Phase 4: Safety-Nets** — Added `LastStepSignature` and `ConsecutiveObserveCount` tracking to `ExecutionContext`. Repeat detection: if LLM generates identical tool+params as previous step, injects steering instruction and skips execution. Observe_ui loop detection: if observe_ui called 4+ consecutive times, injects steering instruction.

**Phase 5: Campaign Test Runner** — Added `TestAllCampaigns_ParseAndValidate` (parses all 10 YAMLs, verifies parse + topological order) and `TestAllCampaigns_StartURLValidation` (validates all start_url values are valid HTTP(S) URLs). All 10 campaigns pass.

## What to watch for
- `start_url` is optional — missing field falls back to existing dependency-context regex extraction
- Safety-net thresholds (repeat = 1, observe_ui loop = 4) may need tuning after real LLM runs
- `URLContext()` renders "Current URL" when `CurrentURL` is set, "Start URL" when only `StartURL` is set, "No URL context available" when neither is set — this is by design
- Pre-existing Windows TempDir race condition in engine tests may show intermittently

## Key decisions
- Used structured `start_url` field instead of embedding URLs in prose goals (ChatGPT recommendation). Separates machine-readable config from natural-language intent.
- `start_url` is optional for backward compatibility with existing campaigns
- CurrentURL tracked per-flow after successful navigate, not per-LLM-call
- Safety-nets inject steering instructions (not hard breaks) — LLM can self-correct with context
- Repeat detection compares tool name + all param values; no params means no repeat check (observe_ui with no params is not caught by repeat detection, but observe_ui loop detection handles that case)
