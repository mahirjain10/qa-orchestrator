## Goal
Fix 6 code-review bugs across engine, orchestrator, trace store, and artifact store.

## What happened

### 1. GetEligibleFlows — only returned zero-dependency flows (dependencies.go:149-157)
- **Root cause:** Function only checked `len(flow.DependsOn) == 0`, never looked at runtime state.
- **Fix:** Added `completedFlowIDs map[string]bool` parameter. Iterates each flow's DependsOn and returns it only when all deps are in the completed set. Updated test with 3 scenarios (nil/empty completed, partial, all).
- **Impact:** Dependent flows now become eligible after their upstream completes.

### 2. RunFlowWithRetry — off-by-one in loop bound (engine.go:994-1015)
- **Root cause:** Loop `attempt < maxRetries` gave `maxRetries` total attempts instead of `maxRetries+1`. Caller papered over it with `retryLimit+1`.
- **Fix:** Changed loop to `attempt <= maxRetries`, backoff check to `attempt < maxRetries`. Changed caller to pass `retryLimit` directly.
- **Impact:** `maxRetries=2` now correctly yields 1 initial attempt + 2 retries = 3 total.

### 3. 7 setter methods on AgentEngine lack mutex (engine.go:163-171, 1033-1102)
- **Root cause:** All setters except `SetDependencyContext` wrote fields without `e.mu` protection, causing data races under concurrent access.
- **Fix:** Added `e.mu.Lock()/Unlock()` to `SetLLMClient`, `SetBrowserTools`, `SetTraceStore`, `SetArtifactStore`, `SetLifecycleController`, `SetLifecycleRunID`, `SetToolRegistry`. `SetLifecycleRunID` uses RLock since it only reads `e.lifecycle`.

### 4. Trace store O(n²) I/O on every append (trace/store.go:150-178)
- **Root cause:** `persist()` marshaled ALL events for a runID and wrote to temp+rename on every Append call.
- **Fix:** Added `persistedCount map[string]int` to track last-persisted offset. `persist()` now appends only new events since last persist using `os.OpenFile(path, os.O_APPEND|os.O_CREATE|os.O_WRONLY)`. `loadFromFile()` initializes `persistedCount` from file length.

### 5. Artifact Delete() leaves inconsistent state on partial failure (artifact/store.go:218-238)
- **Root cause:** `Delete()` removed in-memory index entries and called `delete(s.artifacts, runID)` even if file removal failed, leaving orphaned index entries.
- **Fix:** Try removing all files first. If any fail, abort with error without modifying in-memory state. Only after all files are removed, update index + artifacts + persist.

### 6. loadIndex() silently swallows all errors (artifact/store.go:301-317)
- **Root cause:** Both `os.ReadFile` and `json.Unmarshal` errors were discarded with bare `return`.
- **Fix:** Changed signature to `loadIndex() error`. Returns descriptive errors for read/parse failures. `os.IsNotExist` is handled gracefully (returns nil). Both callers (`Get`, `GetByRunID`) now propagate the error.

## What to watch for
- `GetEligibleFlows` now requires a `completedFlowIDs` map — all callers must pass it.
- Trace store persists via append — if the file is truncated externally, `persistedCount` will be out of sync. NOT an issue in normal operation since the store holds the authoritative in-memory state.

## Key decisions
- Used append-over-rewrite for trace store instead of batching — simpler, still O(n) per event.
- Artifact Delete uses a two-phase approach: remove files first, then update state. This is atomic in practice since state updates are lock-protected and happen only after all files succeed.
