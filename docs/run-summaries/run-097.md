### 4. LifecycleController lacks state transition validation (lifecycle.go:125-141)
- **Root cause:** `SetStatus`, `SetWaitingForInput`, and `AcknowledgeInput` allowed arbitrary state transitions regardless of current state.
- **Fix:** Added `transitionValid(from, to)` function with the complete state machine:
  - PENDING → RUNNING
  - RUNNING → PAUSING, CANCELLING, COMPLETED, FAILED, WAITING_FOR_INPUT
  - PAUSING → PAUSED
  - PAUSED → RESUMING, CANCELLING
  - RESUMING → RUNNING
  - WAITING_FOR_INPUT → RUNNING, CANCELLING
  - CANCELLING → CANCELLED
  - CANCELLED, COMPLETED, FAILED → terminal (no outgoing)
- Also wired `transitionValid` into `RequestCancel` (was using a manual switch).
- Invalid transitions are silently ignored (setter is a no-op).

### 5. AcknowledgeCancel doesn't verify Cancelling state (lifecycle.go:93-97)
- **Root cause:** `AcknowledgeCancel()` set status to CANCELLED unconditionally, even if not in CANCELLING state.
- **Fix:** Changed to `AcknowledgeCancel() bool` — returns false if current status is not CANCELLING, only transitions to CANCELLED when valid.
- Updated test to use fresh controllers for each CanCancel scenario to avoid cross-state contamination.
