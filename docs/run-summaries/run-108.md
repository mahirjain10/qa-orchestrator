## What to watch for
- `FilterInput.Blur()` requires the field to be addressable (pointer receiver). Works because `m` is `*MainScreen`.
- The `filterInput` is never passed the enter/escape keys — we explicitly `Blur()` it instead. This is intentional: the enter handler doesn't need to process the key, only reset focus state.

## Key decisions
- **Explicit Blur() over event passthrough:** Rather than pass the enter event to `FilterInput.Update(msg)` and capture the returned model (which worked but returned a stale cmd), we explicitly `Blur()` the textinput. This is cleaner and matches the pattern already used in `handleCommandKey` for the command bar input.
- **Blur() on both enter and escape:** The escape branch also bypasses `FilterInput.Update(msg)`, so it needed Blur() too for consistency.
