## What happened

### Bug 1: Auto BaseURL set before provider detection (config.go:108-111)
- **Root Cause:** When `LLM_PROVIDER=auto`, the `provider` variable was still `"auto"` at the BaseURL default logic. The check `provider != "gemini"` was always true, so BaseURL was set to OpenRouter URL even when the model would later detect as Gemini.
- **Fix:** Eagerly resolve the provider from model name using `detectProviderName()` before deciding the BaseURL default.

### Bug 2: ThinkingBudget parse error silently ignored (config.go:146-149)
- **Root Cause:** `budget, _ = parseInt(budgetStr)` discarded the parse error.
- **Fix:** Check the error and return it, consistent with the `maxRetries` parsing pattern. Also added negative check.

### Bug 3: Gemini drops TopP/StopSequences (provider_gemini.go:139-156)
- **Root Cause:** `geminiGenerationConfig` had `TopP` and `StopSequences` fields but `BuildRequest()` never assigned them from `req.TopP` and `req.Stop`.
- **Fix:** Added `TopP: req.TopP` and `StopSequences: req.Stop` to the config struct literal.

### Bug 4: NewSimpleClient ignores apiKey parameter (client.go:291-307)
- **Root Cause:** When `LoadConfig()` succeeded (reading env vars), the `apiKey` parameter was never used.
- **Fix:** Added `else if apiKey != ""` branch to override `cfg.APIKey` with the passed key.
