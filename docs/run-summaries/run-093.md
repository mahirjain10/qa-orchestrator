## What happened
1. **Bug 1** (`runtime.go:714`): `runWithContext` had goroutine leak. Changed `fn` signature to `func(context.Context) error`, added `ctx.Err()` check in all 17 closures, added channel drain on ctx.Done().
2. **Bug 2** (`registry.go:153`): `checkSelectorExists` silently returned nil on JS eval failure, type assertion failure, and JSON parse failure. All three paths now return descriptive errors.
3. **Bug 3** (`registry.go:251`): `wait_for` called `checkSelectorExists` before `WaitForSelector`, defeating dynamic element detection. Removed the call — wait_for now waits.
4. **Bug 4** (`prompts.go:186`): `sanitizeGoal` didn't escape `</`, allowing XML tag breakout from goal text.
5. **Bug 5** (`planner.go:215`): Steering instructions unsanitized — injected `sanitizeDOM(inst)` into the steering loop.
6. **Bug 6** (`planner.go:113`): `sanitizeDOM` missing `"`, `\`, `\n`, `[`, `]` escapes. Added with correct order (backslash first).
7. **Bug 7** (`config.go:108`): `BaseURL` never wired into providers. Added logic to `OpenRouterProvider`, `OpenAIProvider`, `GeminiProvider` — `Endpoint()` checks BaseURL before default.
8. **Bug 8** (`retry.go:110`): `IsNetworkError`/`IsAPIError` didn't unwrap `*RetryableError`. Added `errors.As` before type assertions.
9. **Bug 9** (`client.go:55`): Fallback models used same provider as primary. `Generate` now calls `DetectProvider(model)` for each fallback and routes via `doRequestWithProvider`.
10. **Bug 10** (`client.go:82`): `Generate` mutated caller's `GenerateRequest` struct. Added `clone := *req` at top.
11. **Bug 11** (`client.go:28`): Dead `retryConfig` field in `HTTPClient` struct. Removed field and its init.
