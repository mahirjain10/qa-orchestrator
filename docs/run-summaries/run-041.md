## What to watch for
- `Truncate()` uses byte length, not rune length — multi-byte unicode characters may produce shorter visual output than maxLen. This is acceptable for terminal rendering where cell width matters more than byte count.
- `SafeWidth()` is used with `util.Truncate()` to prevent negative maxLen values when terminal is very narrow.
- `FormatDurationShort()` uses a custom `itoa()` to avoid importing `strconv` — this is intentional to keep the package dependency-free except for `time`.

## Key decisions
- Added `TruncateStart()` in addition to `Truncate()` and `TruncateMiddle()` because path display needs to keep the filename (end) visible, not the directory prefix.
- Kept `formatBytes()` in `trace_panel.go` — it's domain-specific (artifact sizes) and doesn't belong in a general utility package.
- `FormatDurationShort()` returns "MMmSSs" format — useful for compact status bar display in future phases.
